De quoi parle-t-on?

Produire un document (output) html, PDF, word etc. qui inclue des parties de texte qui peuvent tre formattes, et des bouts de codes R (ou autres), le tout partir d'un document de type Rmd dans R.

Pourquoi utiliser R Markdown?

  • Arrter de faire du copier-coller entre R et un autre document!
  • Envoyer des rsultats un collaborateur
  • Collaborer plus facilement sur un code
  • Pouvoir mettre jour des rapports plus rapidement
  • Monter plus facilement un cours!

R Markdown = Markdown + R

Markdown : Language de balisage lger (!!), avec une syntaxe facile lire et crire.

.md .html ( .doc / .pdf)

Par exemple, mettre un mot en italique:

html : <i>un mot</i>

laTeX: \textit{un mot}

markdown: _un mot_

R Markdown = Markdown + R

R Markdown : Document Markdown, mais avec des "chunks" (bouts) de code R intgrs

.Rmd .md .html ( .doc / .pdf)

.Rmd .R

Exercice #1b - Transformation dans R

Compiler le fichier .Rmd en .html, .md et .R (voir EnvoieDirectDeR.R)

library(knitr)
library(rmarkdown)

# Rpertoire de travail  = endroit o les fichiers .Rmd sont...
setwd("C:/Users/Audrey/Documents/CSBQ/Ateliers/H2018/RMarkDown")

# .Rmd --> .html
rmarkdown::render("CodeSimple.Rmd") # .html

# .Rmd --> .md
knit("CodeSimple.Rmd") # .md

# .Rmd --> .R
purl("CodeSimple.Rmd") # .R
purl("CodeSimple.Rmd", output = " CodeSimple2.R", documentation = 2)
